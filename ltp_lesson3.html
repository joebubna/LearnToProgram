Learning to Program: Lesson #3

Goal:

To understand functions and javascript event binding.

Exercise:

We have two text boxes setup, one for entering your First name, and one for entering your Last name. First, we are going to make it so the names entered in those boxes get combined into a Full name. Then we'll see a really simple example of how we can use a Function to improve our code.

Steps:

  1. A new javascript file has been included on this page in addition to jQuery! Open up "lesson3.js" from the "resources/student/" folder in your text editor - this is where you will be writing your code now instead of using the Console. All the code you are instructed to write for this lesson should be saved into that .js file.

    Below is how your custom javascript file gets included on the web page. <body> ... content here ... ... content here ... <script type="text/javascript" src="resources/jQuery-2.1.4.js"></script> <script type="text/javascript" src="resources/student/lesson3.js"></script> </body>

    IMPORTANT!!!

    Just because you won't be writing your code in the console anymore DOES NOT mean you should stop opening it. When you run into errors that cause your code to not work, the console will usually tell you where to look for the error!!! This is SUPER useful as you WILL make typos, even when following this guide. There's also ways you can walk through your code's execution step-by-step using the "sources" tab in Chrome or "debugger" tab in Microsoft Edge, but that's beyond the scope of this tutorial.
  2. Before we get started with our Javascript code, take a look at the HTML we will be working with from this lesson's Testing Area: <div>First Name: <input type="text" id="firstName"></div> <div>Last Name: <input type="text" id="lastName"></div> <hr> <div>Fullname = <span id="fullName"></span></div>
  3. First, we want to setup our Javascript events that will cause our code to get executed. Events are things such as clicking something on the web page, moving the mouse over an area of the web page, pressing a key on the keyboard, etc. Since we want to "do something" when a user types into either the first or last name textboxes, we will use an event called "oninput".

    Recall that in the last lesson we used the jQuery "append" function to modify the DOM like so: $('#testing ol').append('<li>I ROCK!</li>');
    To attach an event to a DOM item or items, we will use a similar format like this: 
    $('#firstName').on('input', function() {
    ... code to execute when event triggers ...
});
    Type the above code into your lesson3.js file, then we'll fill in the center part later. See below for an explanation.

    Explanation:

    • First, we use a jQuery selector - $('#firstName') - to select the DOM element we want to attach an event to.
    • Second, we are calling the "on" function with the .on() code. This function creates the event attachment, but we have to pass it additional information which in programming lingo are called "parameters."

      The parameters it requires from us are:

      1. The events we want to trigger our code on.
      2. The code we want to execute when a trigger happens, which will be a function.
      Note: The actual values being passed into a function with parameters are referred to as the "arguments." If you have a function that calculates a person's age when given their birth year, "birthYear" would be its parameter and "1985" might be the argument you pass it.
    • Okay, now for the confusing part... the parameters between the "on" parenthesis. So as we just mentioned, there's two arguments being passed. so it might be easy to understand to look at it this way: $('#firstName').on(parameter1, parameter2); WHERE parameter1 = 'input' AND parameter2 = function() { ... code to execute ... }
    • Passing in the string "input" as the first parameter tells the function that the event we want to bind to this DOM element is the oninput event. It's technically possible to pass multiple events seperated by a space (e.g. 'input mouseover click keyup') but we don't need to do that for this example.
    • The second parameter is the doozey, and that's because we're passing in a whole function that we want to execute when the event gets triggered. This is called an anonymous function. Don't worry if it confuses you right now, we'll start digging into functions in a bit. Hang in there!! :)
  4. Alright, let's fill in the code we want to execute when the event triggers. It's a pretty simple two lines, and you can probably figure out what it does right away. Add the following two lines to your code: var fullname = $('#firstName').val() + ' ' + $('#lastName').val(); $('#fullName').text(fullname);

    We're constructing the person's full name by taking the text entered into the firstName box, adding a space, then adding ("concatonating" is technical term) the text in the lastName box. After we got the fullname, we then use jQuery's amazing simplicity to update the text in our fullName field on the page.

    Go ahead and reload this page after you've added the above to your lesson3.js file, then try typing something in the First Name box. Your code in your lesson3.js file should look like the below at this point:

    $('#firstName').on('input', function() { var fullname = $('#firstName').val() + ' ' + $('#lastName').val(); $('#fullName').text(fullname); });
  5. Let's do the same thing for the lastName box. Add the following to your file: $('#lastName').on('input', function() { var fullname = $('#firstName').val() + ' ' + $('#lastName').val(); $('#fullName').text(fullname); }); Now reload the page and test the results again. If you didn't make any typos, you should see the "Fullname" updating instantly when you start typing in either the first or last name boxes.
  6. One thing you may have noticed, is that our code is almost identical for both the firstName and lastName pieces. In-fact, the code we are executing when our events trigger is EXACTLY IDENTICAL!!! 
    $('#firstName').on('input', function() {
    var fullname = $('#firstName').val() + ' ' + $('#lastName').val();
    $('#fullName').text(fullname);
});

$('#lastName').on('input', function() {
    var fullname = $('#firstName').val() + ' ' + $('#lastName').val();
    $('#fullName').text(fullname);
});
    This is a PERFECT opportunity to pull that code out, and place it into a function. Functions serve as reusable blocks of code that help you keep your code clean and organized - reducing repitition by allowing you to execute it multiple times without writing it out each time.
  7. Below the code you've already written, add the following: 
    function outputName() {
    var fullname = $('#firstName').val() + ' ' + $('#lastName').val();
    $('#fullName').text(fullname);
}

    As you can see, it's pretty easy. You just declare that you're writing a function, give the function a name, declare any parameters you require between parenthesis (leave empty if no data is needed), and then finally write a block of code you want to execute by placing it between brackets {}.

  8. Now that we have our code added to a function, go back and edit what we wrote earlier, replacing the repetitive code with calls to our function. The end result of your code for this project should look like this: 
    $('#firstName').on('input', function() {
    outputName();
});

$('#lastName').on('input', function() {
    outputName();
});


///////////////////////////////////////////////////////
// Function(s)
///////////////////////////////////////////////////////

function outputName() {
    var fullname = $('#firstName').val() + ' ' + $('#lastName').val();
    $('#fullName').text(fullname);
}
  9. One quick last note: Now that we moved our repetitive code to the outputName function and all that's left to do when the event triggers is calling outputName() and nothing else, we could eliminate the anonymous function and just call outputName directly like so: 
    $('#firstName').on('input', outputName);
$('#lastName').on('input', outputName);
